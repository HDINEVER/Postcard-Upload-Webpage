🔐 Storage Buckets 权限配置

解决 "Failed to fetch" CORS 错误的关键步骤

⚠️ 当前问题

虽然你已经配置了 Web 平台（localhost），但是 Storage Buckets 的权限没有配置，导致文件上传时出现 CORS 跨域错误。

步骤 1: 登录 Appwrite 控制台

访问你的 Appwrite 控制台：

🔗 打开 Appwrite 控制台

步骤 2: 选择项目

找到并点击项目：69dc5b0700295f5740d0

步骤 3: 配置每个 Storage Bucket 的权限

你需要为 3 个存储桶 分别配置权限：

📸 明信片设计 (Postcard)
ID: 69df42a60004a562ba07
📊 演示文稿演讲 (Presentation)
ID: 69df430100088d621422
🎥 项目上传视频 (Video)
ID: 69df4311003df133de23

对每个存储桶执行以下操作：

  1. 在左侧菜单点击 Storage
  2. 点击对应的存储桶（根据上面的 ID 找到它）
  3. 点击 Settings 标签
  4. 点击 Permissions 子标签
  5. 点击 + Add Role 按钮
  6. 在弹出窗口中配置：
    Role Type: 选择 Any
    Create: 勾选 ✓
    Read: 勾选 ✓
  7. 点击 Create 或 Save 保存
  8. 重复以上步骤，配置其他两个存储桶

步骤 4: （可选）配置 Collection 权限

如果上传时还提示权限错误，也需要配置数据库集合的权限：

  1. 在左侧菜单点击 Databases
  2. 选择数据库 69de43e100124b512cbb
  3. 点击 Collection project
  4. 点击 Settings → Permissions
  5. 添加 Role: Any
  6. 勾选权限：Create 和 Read
  7. 保存

步骤 5: 验证配置

  1. 配置完成后，等待 1-2 分钟让 Appwrite 配置生效
  2. 回到浏览器，强制刷新页面（Ctrl+Shift+R 或 Cmd+Shift+R）
  3. 再次测试上传功能
  4. 打开浏览器控制台（F12）查看是否还有 CORS 错误
