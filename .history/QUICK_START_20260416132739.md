# 📚 Appwrite 集成快速开始指南

## 🎯 概述

本项目已集成 Appwrite 后端服务。您只需完成以下 5 个简单步骤即可启动应用。

---

## 🚀 5 分钟快速开始

### 第 1 步：获取 Appwrite 信息 (2 分钟)

登录你的 Appwrite 控制台

复制以下内容：
1. **Project ID** → Settings → Project ID
2. **Database ID** → 创建或选择数据库
3. **Collection ID** → 选择 "submissions" 集合
4. **Bucket ID** → 存储 → 选择 "submissions" 桶
5. **Appwrite URL** → 通常为 `https://your-domain.com/v1`

### 第 2 步：填写 `.env` 文件 (1 分钟)

编辑项目根目录的 `.env` 文件：

```env
VITE_APPWRITE_URL=https://your-appwrite-domain.com/v1
VITE_APPWRITE_PROJECT_ID=<粘贴你的 Project ID>
VITE_APPWRITE_DATABASE_ID=<粘贴你的 Database ID>
VITE_APPWRITE_COLLECTION_ID=<粘贴你的 Collection ID>
VITE_APPWRITE_BUCKET_ID=<粘贴你的 Bucket ID>
GEMINI_API_KEY=<你现有的 Gemini API Key>
```

说明：前端直连 Appwrite 时不需要 API Key。

### 第 3 步：启动应用 (1 分钟)

```bash
npm run dev
```

打开 `http://localhost:3000`

### 第 4 步：测试 (1 分钟)

1. 点击 **"作品申报入口"** 按钮
2. 填写表单信息
3. 选择一个测试文件
4. 点击 **"确认提交"**
5. 等待成功消息 ✅

### 第 5 步：验证提交 (1 分钟)

在 Appwrite 控制台检查：
- **Collections** → submissions → 查看新记录
- **Storage** → submissions → 查看上传的文件

## 🔧 Appwrite 后端设置

如果您还没有 Appwrite 项目或需要创建新的 Collection/Bucket，按以下步骤操作：

### 创建数据库和集合

```sql
-- 在 Appwrite Studio 中执行

CREATE COLLECTION submissions (
  name: String,
  phone: String,
  school: String,
  student_id: String,
  category: Enum ['postcard', 'presentation', 'video'],
  file_id: String,
  file_name: String,
  file_size: Integer,
  submitted_at: String
);
```

### 创建存储桶

在 Appwrite 控制台：
1. Storage → Create Bucket
2. ID: `submissions`
3. Name: `Submissions`
4. Max File Size: 209,715,200 (200 MB)
5. Create

### 设置权限

确保 Collection 和 Bucket 的权限允许读写：

```json
{
  "read": ["role:all"],
  "write": ["role:all"]
}
```

## 🐛 故障排除

### 问题：显示 "Unauthorized" 错误

**原因**: 认证失败
**解决**:
1. 检查 `.env` 中的 Project ID 是否正确
2. 检查 Appwrite URL 是否正确（应包含 `/v1`）
3. 重启开发服务器: `npm run dev`

### 问题：文件上传失败

**原因**: 存储桶配置问题
**解决**:
1. 验证 Bucket ID 是否正确
2. 检查存储桶大小限制 (应 ≥ 200 MB)
3. 在 Appwrite Settings 中添加您的应用域到 Platforms/CORS
4. 如果你的 Appwrite 域名前面有 Cloudflare Access、WAF 或 Zero Trust，确保 `/v1/*` 没有被拦截

### 问题：表单提交无数据出现

**原因**: 集合配置问题
**解决**:
1. 验证 Collection ID 是否正确
2. 确保 Collection 包含所有必需字段
3. 检查 Collection 权限是否允许当前角色创建文档

### 问题：浏览器控制台 CORS 错误

**原因**: 跨域请求被阻止
**解决**:
1. 登录 Appwrite 控制台
2. Settings → Platforms
3. 添加您的应用域（开发环境通常填 `localhost`）
4. 如果访问 `/v1/health` 会跳到 Cloudflare Access 登录页，先取消这层拦截或给 `/v1/*` 放行
